Description

Le projet IFRI_comotorage consiste à réaliser une application web qui met en relation les étudiants de l’IFRI souhaitant partager leurs trajets quotidiens entre leur domicile et le campus. Dénommé IFRI_comotorage, chaque utilisateur crée un profil (point de départ habituel, horaires, etc.) et peut publier ou rechercher des offres / demandes de covoiturage. Un algorithme de matching — basé sur la proximité géographique et la compatibilité horaire — propose automatiquement les combinaisons conducteur- passager les plus pertinentes, tandis qu’une messagerie intégrée permet d’affiner les détails du rendez-vous.L’application devra disposer d’interfaces graphiques web communiquant avec une base de données suivant le modèle client-serveur. Elle devra disposer des modules qui suivent : gestion des comptes et profils utilisateur , mise en correspondance des conducteurs et passagers , messagerie instantanée. Le projet est structuré de manière modulaire, avec des composants distincts pour la gestion des utilisateurs, la recherche de trajets, et la messagerie. Chaque module est développé en utilisant Django pour le backend et Bootstrap pour le frontend, assurant ainsi une interface utilisateur réactive et intuitive. La base de données MySQL est utilisée pour stocker les informations des utilisateurs, les trajets, et les messages échangés.
Le projet contient donc trois applications principales : l'application comptes pour les connexions et inscriptions, l'application trajets pour la gestion des trajets et l'application discussions pour la communication entre les utilisateurs. Chaque application est développée en suivant les principes du modèle MVC (Modèle-Vue-Contrôleur), assurant ainsi une séparation claire des préoccupations et facilitant la maintenance et l'évolution de l'application.

    • 🗃️ Modèle de base de données relationnelle

    La base de données suit une structure logique adaptée au domaine du covoiturage.
    Elle est conçue pour stocker les informations des utilisateurs, les trajets, les réservations et les messages échangés entre les utilisateurs.
    Chaque table est liée par des clés étrangères pour assurer l'intégrité des données et faciliter les requêtes complexes.
    Voici un aperçu de la structure de la base de données des tâbles principales utilisées dans le projet IFRI_comotorage :

    Table Decription
    Utilisateurs Informations des membres (nom, rôle, email, mot de passe hashé, etc.)
    Trajets Informations sur les trajets proposés (lieu départ, lieu arrivée, date, heure, conducteur_id)
    Réservations Lien entre passagers et trajets réservés
    Messagerie Messagerie instantanée entre passager et conducteur

    🔐 Gestion des comptes & rôles

    Le projet gère les comptes utilisateurs avec des rôles distincts (conducteur, passager).
    Chaque utilisateur peut créer un compte, se connecter et gérer son profil.
    Les conducteurs peuvent publier des trajets, tandis que les passagers peuvent rechercher et réserver des trajets.
    La sécurité est assurée par le hachage des mots de passe et la validation des entrées utilisateur. De plus , la base est renforcée par des fonctions de contrôles de doublons d'identifiants et de vérification des données saisies lors de l'inscription.
    Un choix de connexion avec votre compte Google est aussi disponible pour une meilleure expérience utilisateur.

    💬 Messagerie instantanée

    Le projet intègre une messagerie instantanée pour faciliter la communication entre les utilisateurs.
    Les passagers peuvent envoyer des messages aux conducteurs pour discuter des détails du trajet, poser des questions ou confirmer leur réservation.
    La messagerie est sécurisée et permet de conserver un historique des échanges pour chaque trajet.
    Cette fonctionnalité améliore l'expérience utilisateur en permettant une communication directe et rapide entre les parties concernées.

    ⚙️Fonctionnement du Matching

    Le projet utilise un algorithme de matching pour associer les conducteurs et les passagers en fonction de leur proximité géographique et de leurs horaires.
    Lorsqu'un conducteur publie un trajet, l'algorithme recherche les passagers dont les points de départ et d'arrivée sont compatibles avec le trajet proposé.
    Les passagers peuvent également rechercher des trajets en fonction de leurs préférences (lieu, heure, etc.).
    L'algorithme prend en compte les préférences des utilisateurs pour proposer les correspondances les plus pertinentes.
    Le système de recherche repose sur une comparaison des lieux de départ et d’arrivée renseignés par le passager avec ceux des trajets disponibles en base. L'algorithme applique le lower() aux chaînes de caractères pour éviter les erreurs de casse.

    Instructions

    Dans cette rubrique nous vous montrerons ou plutôt guiderons à travers le déploiement de notre projet.

    Prérequis pour le déploiement

    1. Installer Python sur son système est crucial pour exécuter le projet IFRI_comotorage.
      • Pour Windows, télécharger l'installeur depuis le site officiel de Python. Pour le telechargement, cliquez sur ce lien. Assurer vous d'installer la version Python 3.13.2 ou installer une version plus récente
      • Pour Linux, utiliser le gestionnaire de paquets (apt, yum, etc.) pour installer Python. Avec linux , l'installation s'avère plus facile il vous suffit de taper la commande suivante dans votre terminal : sudo apt install python3 en étant sur Ubuntu/Debian ou sudo yum install python3 en ulisant d'autres distributions linux tels que Cent OS ; Red Hat , etc .
      • Pour MacOS, utiliser Homebrew ou télécharger l'installeur depuis le site officiel. Assurer vous d'installer la version Python 3.13.2 ou installer une version récente sur le le site officiel

    2. Installer pip (Python package manager)
      Inclus automatiquement avec Python. Vérifiez avec : pip --version

    3. Installer MySQL sur son système localement .
      • Pour les utilisateurs utilsant Windows rendez vous sur le site de télechargement officiel ou installer directement la version mysql Ver 8.0.28 for Win64 on x86_64 (MySQL Community Server - GPL) . Lors de l'installation choissisez le mode par défaut et créer et notez le mot de passe pour l'utilisateur root . De plus assurez vous de mettre dans votre path (système d'environnemnent ) le shell de sql . Ouvrez ensuite votre invite de commandes et lancer SQL avec la commande mysql -u root -p et entrez votre mot de passe pour vous connecter .
      • Pour les utilisateurs de Linux , ouvrez juste un terminal et tapez la commande sudo apt install mysql-server mysql-client après avoir exécuter la commande sudo apt update pour de quelconques mises à jour . Démarrez ensuite votre service MySQL avec la commande sudo systemctl start mysql . Sécurisez le avec sudo mysql_secure_installation et connectez vous avec la commande sudo mysql -u root -p .
      • Pour les utilisateurs de MacOS , avec Homebrew ouvrez votre invite de commandes et tapez la commande brew install mysql . Lancer le avec la commande brew services start mysql. Connectez vous au service MySQL en tapant la commande mysql -u root

    4. Avoir installé Git pour cloner le dépôt du projet.
      Pour l'installation de Git, vous pouvez suivre les instructions sur le site officiel de Git : Installation de Git en installant le paquet de votre système.

    Clonage du projet

    Pour clonez le projet ouvrez juste git installé plus haut et tapez la commande
    git clone https://github.com/lenurb-123/PIL1_2425_26.git

    Tapez cd PIL1_2425_26 pour accéder au repertoire du dossier

    Création de l'environnement virtuel

    Pour créer l'environnement virtuel , assurez-vous d'être dans le répertoire du projet cloné et exécutez la commande suivante :
    python -m venv venv

    Activation de l'environnement virtuel

    Pour activer l'environnement virtuel, utilisez la commande suivante :
    source venv/bin/activate (Linux/Mac) ou venv\Scripts\activate (Windows).
    Une fois l'environnement virtuel activé, vous verrez le nom de l'environnement virtuel dans votre invite de commande, indiquant que vous travaillez dans cet environnement isolé.
    L'activation de l'environnement virtuel est essentielle pour s'assurer que les dépendances du projet sont installées et utilisées correctement, sans interférer avec d'autres projets Python sur votre système.
    Vous pouvez désactiver l'environnement virtuel en exécutant la commande deactivate. Cela vous ramènera à votre environnement Python global.
    Assurez-vous d'activer l'environnement virtuel chaque fois que vous travaillez sur le projet pour garantir que les dépendances sont correctement gérées

    Installation des dépendances

    Pour installer les dépendances du projet, assurez-vous d'être dans le répertoire du projet cloné et de l'environnement virtuel activé. Ensuite, exécutez la commande suivante :
    pip install -r requirements.txt

    Cette commande installera toutes les bibliothèques nécessaires au bon fonctionnement du projet.
    Assurez-vous que l'environnement virtuel est activé avant d'exécuter cette commande. Pour activer l'environnement virtuel, utilisez la commande suivante :
    source venv/bin/activate (Linux/Mac) ou venv\Scripts\activate (Windows).
    Une fois l'environnement virtuel activé, vous pouvez exécuter la commande d'installation des dépendances.

    Import de la base de données

    Pour importer la base de données, vous avez deux méthodes :

    Methode 1 : Via le Terminal SQL avec le fichier structure.sql
    • Ouvrez un l'invite de commandes et connectez-vous à MySQL :
    • mysql -u root -p
    • Créez la base de données :
    • CREATE DATABASE ifri_comotorage;
    • Assurez-vous d'être dans le répertoire du projet dans votre terminal (invite de commandes). C'est-à-dire dans votre projet cloné.
      Une fois dans le répertoire du projet, exécutez la commande suivante pour importer la structure de la base de données :
    • mysql -u root -p ifri_comotorage < db/structure.sql

      • Extrait de settings.py :

      DATABASES = {
                          'default': {
                            'ENGINE': 'django.db.backends.mysql',
                            'NAME': 'ifri_comotorage',
                            'USER': 'root',
                            'PASSWORD': 'votre_mot_de_passe',  # Si vous avez un mot de passe
                            'HOST': 'localhost',
                            'PORT': '3306',
                          }
                        }
    Méthode 2 : Via MySQL Workbench :
    • Ouvrez MySQL Workbench et connectez-vous à votre serveur MySQL.
    • Créez une nouvelle base de données nommée ifri_comotorage.

      • Extrait de settings.py :

      DATABASES = {
                              'default': {
                                'ENGINE': 'django.db.backends.mysql',
                                'NAME': 'ifri_comotorage',
                                'USER': 'root',
                                'PASSWORD': 'votre_mot_de_passe',  # Si vous avez un mot de passe
                                'HOST': 'localhost',
                                'PORT': '3306',
                              }
                            }

    • Puis exécuter :

      python manage.py migrate pour exéuter les migrations
       python manage.py createsuperuser pour la création d'un super utilsateur

    Lancement du projet

    Pour lancer le projet, assurez-vous d'être dans le répertoire du projet cloné et de l'environnement virtuel activé. Ensuite, exécutez la commande suivante :
    python manage.py runserver

    Cette commande démarrera le serveur de développement Django. Vous devriez voir un message indiquant que le serveur est en cours d'exécution sur http://127.0.1:8000/.
    Vous pouvez accéder à l'application en ouvrant un navigateur web et en naviguant vers cette adresse.
